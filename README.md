# Конституция Orgrimmar

Верхнеуровневые правила для всех агентов Family Office AI.

## Что это

Read-only свод правил, который агенты обязаны соблюдать. Конституция стоит выше любых локальных файлов агентов (CONVENTIONS.md, AGENTS.md, SOUL.md). Выше конституции -- только прямой приказ принца.

## Структура

| Файл | Назначение | Частота изменений |
|------|-----------|-------------------|
| `PRINCIPLES.md` | Ядро: миссия, 4 приоритета, фундаментальные принципы | Редко |
| `CHARTER.md` | Устав: роли, границы, координация, эскалация, процессы | По мере необходимости |
| `CHANGELOG.md` | История всех изменений | При каждом изменении |
| `proposals/TEMPLATE.md` | Шаблон для предложений от агентов | Не меняется |

## Иерархия документов (сверху вниз)

```
1. Прямой приказ принца
2. PRINCIPLES.md (ядро конституции)
3. CHARTER.md (устав)
4. CONVENTIONS.md (правила кода и процессов)
5. AGENTS.md (индивидуальное поведение)
6. SOUL.md (характер)
```

## Доступ

- **Принц** -- единственный кто мержит изменения в main
- **Агенты** -- читают при старте сессии, предлагают изменения через PR

## Синхронизация

Каждый сервер: `/home/openclaw/.openclaw/constitution/`

## Как предложить изменение

1. Создай ветку
2. Заполни шаблон из `proposals/TEMPLATE.md`
3. Открой Pull Request
4. Принц ревьюит и мержит (или отклоняет)
