## Core тезис
За последние ~2 месяца (особенно с декабря) программирование изменилось не постепенно, а скачком: coding agents из «почти не работают» перешли в рабочее состояние для длинных, сложных задач.

## Что утверждает автор
- Модели лучше держат длинный контекст, качество и «упёртость» в решении задач.
- Агент может автономно делать end-to-end workflow (настройка сервера, деплой, отладка, отчёт) за ~30 минут.
- Классический режим «сам пишу код в редакторе» быстро уступает режиму «ставлю задачу агенту на английском, управляю и ревьюю параллельно».
- Главный рычаг – подниматься по слоям абстракции и строить оркестрацию агентов (инструменты, память, инструкции, параллельные инстансы).

## Ограничения, которые он прямо отмечает
- Это не магия: нужны направление, вкус, контроль, итерации.
- Лучше работает в чётко формализуемых задачах, где результат можно протестировать/проверить.
- Ключевая компетенция – правильно декомпозировать задачу для делегирования агенту.

## Потенциальные углы для контента
- «Мы перешли из режима Copilot в режим Orchestrator».
- «Побеждает не тот, кто быстрее пишет код, а кто быстрее строит агентные процессы».
